# t7 verify — Concepts wall + slide-in composer

Manual/Playwright checklist for the orchestrator. Exercises: the concept wall
render, the steer flow from the Wall viewer (`?steer=<artistId>`), the
slide-in composer's generate/copy-prompt/paste-back paths, and draft
persistence.

## Setup

1. Start a local-backend dev server (never point Playwright at the real
   Supabase origin):
   ```
   VITE_BACKEND=local npm run dev -- --port 5184
   ```
2. Navigate to `http://localhost:5184/`.
3. Clear any stale service worker / cache before asserting anything:
   ```js
   const regs = await navigator.serviceWorker.getRegistrations()
   await Promise.all(regs.map((r) => r.unregister()))
   const keys = await caches.keys()
   await Promise.all(keys.map((k) => caches.delete(k)))
   ```
   Reload after clearing.
4. Seed a fake local session:
   ```js
   localStorage.setItem('tattoo_local_session', JSON.stringify({
     user: { id: 'local-test-user', email: 'me@pritesh.net' },
   }))
   ```
   Reload. You should land on the Wall (`/`).
5. Clear any leftover composer draft before starting the checklist:
   ```js
   localStorage.removeItem('tattoo_composer_draft')
   ```

## Route + empty state

6. Navigate to `/concepts` directly. **Expect**: hairline bar (wordmark
   "Sable.", Artists/Concepts tabs — Concepts active — "+ New concept"
   button). With no saved concepts, an empty state reads "No concepts yet —
   describe an idea to start the wall." with its own "+ New concept" CTA.

## Steer flow from the Wall viewer (the `G` handoff)

7. Go to `/`, click a wall image to open the full-screen `WallViewer`, note
   the artist name shown.
8. Press `G` (or click "Generate a concept in this style").
   - **Expect**: navigates to `/concepts?steer=<artistId>` and the slide-in
     composer is already open, pre-steered — the steer card shows that
     artist's thumbnail, name, and styles, with copy reading "Steering is
     pre-set to …".
9. Repeat with an artist whose id contains a dot (e.g. `zoia.ink` if seeded)
   — confirm the composer still resolves and displays that artist correctly
   (query params with dots aren't mangled).

## Composer — steer picker

10. Click "change" on the steer card. **Expect**: a scrollable artist list
    drops down, including a "No style steering" option at the top.
11. Pick a different artist — steer card updates immediately. Pick "No style
    steering" — steer card reverts to the unsteered state.

## Composer — generate image path (key configured)

12. Expand "AI setup" and paste a Gemini or OpenAI key, Save.
    **Expect**: the button label changes to "AI setup (key set)".
13. Type an idea into "Your idea", optionally pick a placement chip, click
    "Generate image".
    - **Expect**: button shows "Generating…", then on success the composer
      closes, the draft clears, and a new tile appears on the concept wall
      with a hover caption showing the idea text (or "steered · <artist>" if
      steered).
14. Open the new concept (click its wall tile). **Expect**: the full-screen
    `ConceptViewer` opens — image large-centered, title bottom-left, index
    `01 / N` top-right, prev/next chevrons (if more than one concept),
    "Variants & STL export" button bottom-right with an `I` kbd hint, and a
    "Delete" button. Press `I` (or click the button) — an info panel slides
    in from the right with the full prompt, a style-tag picker, and an
    "AI Results" section (variants + "Add Result").

## Composer — copy-prompt + paste-back path (no key / manual)

15. Remove any saved keys (AI setup → Remove key) so "Generate image" falls
    back to opening AI setup instead of generating.
16. Open "+ New concept", type an idea, click "Copy prompt instead".
    - **Expect**: label flips to "Copied ✓" momentarily; the composer stays
      open (does **not** close or create a concept yet) so the paste-back
      zone is still available.
17. In the composer's "Paste a result back" drop zone, use "Choose file" to
    pick any image file (or drag one onto the zone).
    - **Expect**: the composer closes, the draft clears, and a new tile
      lands on the wall using the same shape as a generated concept
      (title/steer caption, openable in `ConceptViewer`, variants/STL still
      reachable there).

## Prompt packs (secondary disclosure)

18. Open "+ New concept", expand "+ Prompt packs". **Expect**: the existing
    `PromptPackComposer` (free text / Brief idea source, provider-specific
    prompt tabs, "Save Pack") renders inline.
19. Generate and Save a pack. **Expect**: since a prompt pack has no image
    yet, it appears in a "Drafts — awaiting an image" section below the wall
    (not on the wall itself) with its saved prompt pack, an "AI Results"
    variants section, and a "Delete" action.
20. Paste an image into the composer's drop zone right after saving a pack
    (before opening a new concept) — **expect** it attaches to that pending
    draft (moves it out of Drafts and onto the wall) rather than creating a
    second concept.

## Draft persistence

21. Open "+ New concept", type an idea, pick a placement, steer to an
    artist — do **not** generate/copy/paste. Reload the page.
    - **Expect**: reopening "+ New concept" restores the same idea text,
      placement, and steered artist (read from `localStorage
      tattoo_composer_draft`).
22. Confirm `tattoo_composer_draft` is absent from any per-user sync payload
    — it's device-local only, same class as `tattoo_theme`.
23. Successfully generate/copy-paste/paste-back a concept (per steps 13 or
    17) — confirm `localStorage.getItem('tattoo_composer_draft')` is `null`
    afterwards (draft cleared on save).

## Regression checks

24. Confirm legacy concepts (saved before this change, with `imageUrl` set)
    still render on the wall and open correctly in `ConceptViewer`.
25. Confirm the guard in `src/test/v2-tokens.spec.js` still passes — no new
    file under `src/components`/`src/pages` uses the banned
    `cream-muted/30`/`cream-muted/40` utilities.
