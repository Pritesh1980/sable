# t9 verify — WebGL crossfade transition layer

Manual/Playwright checklist for the orchestrator. Exercises: the WebGL
crossfade/ripple in the full-screen viewers, the `?gl=off` and reduced-motion
fallbacks to the CSS path, and that the Top-5 coverflow (now on the same gl
gate) is unaffected. Fonts-offline is a bonus check.

## Setup

1. Start a local-backend dev server (never point Playwright at the real Supabase
   origin):
   ```
   VITE_BACKEND=local npm run dev -- --port 5186
   ```
2. Navigate to `http://localhost:5186/`.
3. Clear any stale service worker / cache before asserting:
   ```js
   const regs = await navigator.serviceWorker.getRegistrations()
   await Promise.all(regs.map((r) => r.unregister()))
   const keys = await caches.keys()
   await Promise.all(keys.map((k) => caches.delete(k)))
   ```
   Reload after clearing.
4. Seed a fake local session so you land on the Wall (`/`):
   ```js
   localStorage.setItem('tattoo_local_session', JSON.stringify({
     user: { id: 'local-test-user', email: 'me@pritesh.net' },
   }))
   ```
   Reload.

## WebGL transitions in the wall viewer (default path)

5. On the Wall, click any image to open the full-screen `WallViewer`.
   - **Expect**: the image fills the stage. Inspect the DOM — the stage is a
     `<canvas>` (WebGL), not an `<img>`, confirming the webgl branch is live.
   - If your GPU/browser has no WebGL, you'll see the `<img>` fallback instead;
     force the CSS path test below to verify that path explicitly.
6. Press `→` / `←` to move within the artist's images.
   - **Expect**: each change **crossfades** with a brief (~600ms) radial ripple,
     not a hard cut. The artwork is letterboxed (never cropped) against the ink
     background.
7. Press `↑` / `↓` to cross to the next/previous artist.
   - **Expect**: same crossfade; the artist plate, index label and filmstrip
     (the HUD) update exactly as before and fade out after ~2s of stillness.
8. Confirm all HUD/keyboard behaviour is identical to the CSS path: `G` →
   `/concepts` steer, `I` → info panel (status/notes/linked ideas), `Esc` →
   back to the wall, `⌘V` paste-to-current-artist, filmstrip click jumps.
9. Console check: no repeated WebGL warnings, no "too many active WebGL
   contexts". Open and close the viewer ~10 times, then run
   `performance.memory` (Chrome) or watch the tab's memory — it should not climb
   unbounded (contexts are disposed on close).

## `?gl=off` forces the CSS path

10. Navigate to `http://localhost:5186/?gl=off` (re-seed session if needed).
    Open the viewer on any image.
    - **Expect**: the stage is a plain `<img>` (object-contain), transitions are
      the CSS `animate-fade-in` cut — no canvas. Everything else (arrows, HUD,
      G/I/Esc, paste) works identically.
11. Alternatively set `localStorage.setItem('tattoo_gl_override', 'off')` and
    reload — same CSS-path result.

## Reduced motion forces the CSS path

12. In DevTools, emulate `prefers-reduced-motion: reduce` (Rendering tab →
    "Emulate CSS media feature prefers-reduced-motion"). Reload, open the viewer.
    - **Expect**: CSS `<img>` path, no WebGL canvas, viewer fully functional.

## Concept viewer parity

13. Go to `/concepts`, generate/seed at least two image concepts, open one full
    screen. `←`/`→` between concepts.
    - **Expect**: same WebGL crossfade (default path) / CSS cut (`?gl=off`); the
      variants & STL panel (`I`), delete, and `Esc` all behave as before.

## Top-5 coverflow unaffected (consolidation regression)

14. Go to `/pipeline` (Home). Rank ≥1 artist so the Top-5 hero shows.
    - **Expect (default)**: the three.js coverflow renders and animates as
      before — drag to spin, tap to focus, drop-out/pull-in buttons work.
    - **Expect (`?gl=off` or reduced-motion)**: the static filmstrip fallback
      renders instead (the coverflow now shares the same gl gate, so these
      overrides also disable it — intended).

## Fonts work offline (SW runtime cache)

15. With the SW registered (reload once online so it installs `tattoo-v3`), go
    offline (DevTools → Network → Offline) and hard-reload.
    - **Expect**: the app still renders in Playfair Display / DM Sans / etc.,
      not a system fallback — the Google Fonts CSS + woff2 were runtime-cached.
16. In Application → Cache Storage → `tattoo-v3`, confirm entries for
    `fonts.googleapis.com` and `fonts.gstatic.com`, plus the hashed
    `three`/coverflow JS chunk.

## Regression checks

17. `npm test -- --run` is green (mode wiring, engine disposal, sw contract, and
    all pre-existing suites).
18. `npm run build` succeeds; note a separate `three` chunk in the output that
    is **not** part of the main `index` bundle.
19. `src/test/v2-tokens.spec.js` still passes (no banned low-opacity utilities in
    the new components).
