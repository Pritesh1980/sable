# t8 verify — quick add-artist + recent drop-zone

Manual/Playwright checklist for the orchestrator. Exercises: the quick-add
modal from the Wall bar, the ≤2-click add flow, image drop-zones on the wall
and the full-screen viewer, the recent (isRecent) dot, and the duplicate-handle
guard.

## Setup

1. Start a local-backend dev server (never point Playwright at the real
   Supabase origin):
   ```
   VITE_BACKEND=local npm run dev -- --port 5185
   ```
2. Navigate to `http://localhost:5185/`.
3. Clear any stale service worker / cache before asserting anything:
   ```js
   const regs = await navigator.serviceWorker.getRegistrations()
   await Promise.all(regs.map((r) => r.unregister()))
   const keys = await caches.keys()
   await Promise.all(keys.map((k) => caches.delete(k)))
   ```
   Reload after clearing.
4. Seed a fake local session:
   ```js
   localStorage.setItem('tattoo_local_session', JSON.stringify({
     user: { id: 'local-test-user', email: 'me@pritesh.net' },
   }))
   ```
   Reload. You should land on the Wall (`/`).

## Quick-add modal — ≤2 clicks

5. Click "+ Add artist" in the hairline bar.
   - **Expect**: a compact v2-styled modal opens over the wall — heading "Add
     an artist", Instagram field (required), display name field, style-tag
     pills, a "Drop images here, or paste (⌘V)" zone, and a footer with "Full
     manage view" (left) and Cancel/Save (right).
6. Type `https://www.instagram.com/some_new_artist/` into the Instagram
   field. **Expect**: no visible normalisation feedback yet — normalisation
   happens on save.
7. Click one or two style-tag pills — **expect** they highlight in
   `v2-accent`.
8. Click "Save" (this is click #2 overall — + Add artist, then Save).
   - **Expect**: the modal closes and the wall re-renders. If the artist had
     no images, it contributes nothing new to the wall yet (no images to
     render) — confirm via `/gallery?mode=manage` that `@some_new_artist` now
     exists with the chosen tags.

## Quick-add with images staged

9. Click "+ Add artist" again. Fill in a fresh handle (e.g. `@another_new`).
10. Drag an image file onto the drop zone — **expect** the zone highlights
    (`v2-accent` border + tint) while dragging, and a thumbnail preview
    appears after drop, with a small "×" remove button.
11. Click "Save". **Expect**: the modal closes, and the wall now shows a new
    piece for `@another_new` with a **recent dot** (small accent-coloured
    dot, top-right of the image) — since it was just added.

## Duplicate-handle guard

12. Click "+ Add artist", type the handle of an artist that already exists
    (e.g. `@zoia.ink` if seeded).
    - **Expect**: "Save" becomes disabled; a message appears — "@zoia.ink is
      already in your collection." — with a button "Add images to … instead".
13. Drop an image onto the drop zone, then click "Add images to … instead".
    - **Expect**: the modal closes, no duplicate artist is created (confirm
      via `/gallery?mode=manage` row count unchanged), and the dropped image
      now appears among that artist's pieces on the wall with a recent dot.

## Drop-zone #1 — masonry wall piece

14. On the wall (masonry grid, not the full-screen viewer), drag an image
    file over any existing artist's piece.
    - **Expect**: that piece highlights with a `v2-accent` ring while
      dragging over it; other pieces are unaffected.
15. Drop the file. **Expect**: the ring highlight clears, and shortly after,
    a **new piece** for that same artist appears on the wall with a recent
    dot. Confirm it was added to the correct artist (open the piece — the
    artist plate should match the one you dropped onto).

## Drop-zone #2 — paste in the full-screen viewer

16. Click any wall piece to open the full-screen `WallViewer`.
17. Copy an image to the OS clipboard (e.g. screenshot, or copy an image from
    a browser tab), then paste (⌘V) while the viewer is open.
    - **Expect**: after a short delay the pasted image is added to the
      **artist currently in view** (not whichever artist you started from,
      if you've navigated with ←/→/↑/↓ since opening).
18. Navigate to a different artist within the viewer (↑/↓), then paste again.
    - **Expect**: the new image attaches to the artist now in view, not the
      original one.
19. Close the viewer (Esc) and confirm both pasted images now show recent
    dots on the wall, under their respective artists.

## Recent dot survives reload

20. Reload the page fully (hard refresh). **Expect**: all images added in
    this session still show recent dots (addedAt persisted, not lost on
    reload) — confirm by checking the wall immediately after reload, before
    the 14-day window would ever lapse.
21. Open dev tools → Application → Local Storage → `tattoo_artists_meta` —
    confirm the newly added artists/images appear as small canonical refs
    (`{ "key": ... , "addedAt": ... }` or `{ "url": ..., "addedAt": ... }`),
    never as raw base64 `data:` URLs.

## Regression checks

22. Confirm the old `/gallery?mode=manage` route still works standalone (full
    add-artist form, maintenance table) — reachable via the modal's "Full
    manage view" link, which also closes the quick-add modal.
23. Confirm `src/test/v2-tokens.spec.js` still passes — no new file under
    `src/components`/`src/pages` uses the banned `cream-muted/30`/
    `cream-muted/40` utilities.
24. Confirm keyboard navigation (←/→/↑/↓/G/I/Esc) in the `WallViewer` still
    works unaffected by the new paste listener.
